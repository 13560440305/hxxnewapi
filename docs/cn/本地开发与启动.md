# 本地开发与启动

本文说明在 Windows / macOS / Linux 上本地运行 hxxnewapi 服务。

## 环境要求

| 组件 | 版本要求 |
|------|----------|
| Go | ≥ 1.22 |
| Bun | 最新稳定版（前端开发/构建） |
| MySQL | ≥ 5.7.8（推荐 8.x） |
| Redis | 任意稳定版（启用密码时需在 `.env` 中配置） |

未配置 `SQL_DSN` 时，程序会回退到 SQLite（`one-api.db`）。未配置 `REDIS_CONN_STRING` 时，Redis 相关功能不可用。

## 1. 配置环境变量

在项目根目录复制示例配置并编辑：

```bash
cp .env.example .env
```

常用项（MySQL + Redis 示例）：

```env
# 数据库（MySQL）
SQL_DSN=root:your_password@tcp(127.0.0.1:3306)/hxxnewapi?parseTime=true

# Redis（密码含 # 等特殊字符需 URL 编码，如 # → %23）
REDIS_CONN_STRING=redis://:your_password@127.0.0.1:6379/0

# 可选：监听端口，默认 3000
# PORT=3000
```

启动时会自动加载项目根目录下的 `.env` 文件。

### 创建 MySQL 数据库

首次使用前在 MySQL 中执行：

```sql
CREATE DATABASE hxxnewapi CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

## 2. 启动方式

**首次运行必须先构建前端。** 后端使用 `go:embed web/dist` 嵌入静态资源，若 `web/dist` 不存在，执行 `go run main.go` 会报错：

```text
pattern web/dist: no matching files found
```

在项目根目录执行：

```bash
cd web
bun install
bun run build
cd ..
```

### 方式 A：仅后端（推荐日常开发）

适用于已构建过前端，或只需调试 API / 管理后台（使用内嵌的 `web/dist`）。

```bash
# 项目根目录
go run main.go
```

若 `web/dist` 不存在或需更新前端，先执行：

```bash
cd web && bun install && bun run build
cd ..
go run main.go
```

访问：**http://localhost:3000**

### 方式 B：前后端分离开发

后端与前端分别启动，前端热更新，API 代理到后端。

**终端 1 — 后端：**

```bash
go run main.go
```

**终端 2 — 前端：**

```bash
cd web
bun install
bun run dev
```

访问：**http://localhost:5173**（Vite 会将 `/api` 等请求代理到 `http://localhost:3000`）

### 方式 C：编译为单文件可执行程序

将前端打包进二进制，适合本地验收或部署前测试：

```bash
make build
```

Windows 生成 `new-api.exe`，Linux/macOS 生成 `new-api`：

```bash
# Windows
.\new-api.exe

# Linux / macOS
./new-api
```

## 3. 首次登录

数据库为空且尚无用户时，程序会自动创建 root 管理员：

| 字段 | 默认值 |
|------|--------|
| 用户名 | `root` |
| 密码 | `123456` |

登录后请立即修改密码。

## 4. 常用命令速查

| 场景 | 命令 |
|------|------|
| 安装前端依赖 | `cd web && bun install` |
| 构建前端 | `cd web && bun run build` |
| 开发运行后端 | `go run main.go` |
| 开发运行前端 | `cd web && bun run dev` |
| 一键构建（前端+后端） | `make build` |
| 编译后端（不构建前端） | `go build -o new-api .` |

## 5. Docker 部署

容器化部署见项目根目录 [README.md](../../README.md) 第一节「Docker 容器编译与运行」，或使用：

```bash
docker-compose up -d --build
```

## 6. 故障排查

| 现象 | 处理建议 |
|------|----------|
| `Redis ping test failed` | 检查 Redis 是否启动、`REDIS_CONN_STRING` 密码是否正确（含 `#` 需 `%23` 编码） |
| MySQL 连接失败 | 确认库 `hxxnewapi` 已创建、账号密码与端口正确 |
| 页面空白或 404 | 执行 `cd web && bun run build` 后再启动后端 |
| `pattern web/dist: no matching files found` | 同上，需先构建前端生成 `web/dist` |
| 端口被占用 | 在 `.env` 中设置 `PORT=其他端口` |

更多环境变量说明见项目根目录 `.env.example`。
