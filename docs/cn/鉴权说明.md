# 鉴权说明

本页说明后台管理接口的鉴权方式与权限级别。

---

## 说明

后台管理接口采用多级鉴权机制，常见级别为：**公开**、**用户**、**管理员**、**Root**。

---

## 认证方式（二选一）

### Session

通过登录接口获取 Session 后，由客户端维护会话（如 Cookie）。

**登录接口**：`POST /api/user/login`

### Access Token（推荐）

在请求头中携带访问令牌：

```
Authorization: Bearer {token}
```

Token 可在「个人设置 - 安全设置 - 系统访问令牌」中生成。

---

## 必需请求头

部分接口要求携带用户标识请求头：

```
New-Api-User: {user_id}
```

其中 `{user_id}` 必须与当前登录用户一致，否则无法通过校验。

---

## 权限级别

| 级别 | 说明 |
|------|------|
| **公开（Public）** | 无需鉴权即可访问 |
| **用户（User）** | 需要已登录或携带有效 Access Token |
| **管理员（Admin）** | 需要具备管理员权限 |
| **Root** | 最高权限，仅 Root 账号可用 |

调用管理接口时，请根据接口要求的权限级别使用对应身份的 Session 或 Token。
